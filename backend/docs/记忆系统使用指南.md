# DeerFlow 记忆系统使用指南

> 完整的长期记忆解决方案，让 AI Agent 记住用户的偏好、背景和对话历史

---

## 📚 什么是记忆系统？

DeerFlow 的记忆系统是一个**全局的长期记忆机制**，它能够：

1. **自动记录**：在对话结束后，使用 LLM 自动提取关键信息
2. **智能存储**：将用户的上下文、偏好、知识点结构化存储
3. **动态注入**：在后续对话中，自动注入相关的记忆内容
4. **持续更新**：随着对话的进行，不断更新和完善记忆

**效果**：让 AI Agent 像真人一样，能够记住之前的对话，了解用户的背景和偏好，提供更加个性化的服务。

## 🚀 快速开始

### 1. 基础使用（3 行代码）

```python
from src.agents.memory import get_memory_data, update_memory_from_conversation
from langchain_core.messages import HumanMessage, AIMessage

# 读取记忆
memory = get_memory_data()
print(f"工作背景: {memory['user']['workContext']['summary']}")

# 更新记忆
messages = [
    HumanMessage(content="我在研究 LangGraph"),
    AIMessage(content="很好！LangGraph 是一个强大的框架...")
]
success = update_memory_from_conversation(messages, thread_id="thread_001")
```

### 2. 查看记忆文件

记忆存储在 JSON 文件中，默认位置：

```bash
# 全局记忆
{base_dir}/memory.json

# 查看内容
cat /path/to/memory.json | jq .

# 查看 facts 数量
cat /path/to/memory.json | jq '.facts | length'
```

### 3. 配置启用

在 `config.yaml` 中：

```yaml
memory:
  enabled: true                  # 启用记忆功能
  debounce_seconds: 30           # 30秒后批量处理更新
  max_facts: 100                 # 最多保存100条事实
  fact_confidence_threshold: 0.7 # 只保存置信度>=0.7的事实
  max_injection_tokens: 2000     # 注入最多2000个tokens
```

## 💾 记忆存储结构

### JSON 文件格式

```json
{
  "version": "1.0",
  "lastUpdated": "2026-04-03T10:30:00.000Z",
  
  "user": {
    "workContext": {
      "summary": "后端工程师，专注于 Python 和 AI 开发，主要项目 DeerFlow",
      "updatedAt": "2026-04-03T10:30:00.000Z"
    },
    "personalContext": {
      "summary": "双语能力（中文/英文），偏好简洁的代码风格",
      "updatedAt": "2026-04-03T10:25:00.000Z"
    },
    "topOfMind": {
      "summary": "正在学习 LangGraph；研究 AI Agent 架构；探索多模态应用",
      "updatedAt": "2026-04-03T10:30:00.000Z"
    }
  },
  
  "history": {
    "recentMonths": {
      "summary": "过去3个月：实现了记忆系统、优化了token计算、集成了MCP服务器",
      "updatedAt": "2026-04-03T10:20:00.000Z"
    },
    "earlierContext": {
      "summary": "3-12个月前：建立了项目架构、开发了技能系统",
      "updatedAt": "2026-03-15T08:00:00.000Z"
    },
    "longTermBackground": {
      "summary": "5年 Python 经验，专注于构建生产级 AI 系统",
      "updatedAt": "2026-02-01T12:00:00.000Z"
    }
  },
  
  "facts": [
    {
      "id": "fact_a1b2c3d4",
      "content": "偏好使用 pytest 进行 Python 测试",
      "category": "preference",
      "confidence": 0.95,
      "createdAt": "2026-03-20T14:30:00.000Z",
      "source": "thread_abc123"
    }
  ]
}
```

### 数据分类

#### User 部分（用户上下文）

| 字段 | 内容 | 示例 |
|------|------|------|
| `workContext` | 工作背景 | "后端工程师，主要项目 DeerFlow，技术栈 Python/TypeScript" |
| `personalContext` | 个人背景 | "双语能力（中英），偏好函数式编程" |
| `topOfMind` | 当前关注点 | "正在学习 LangGraph；研究 Agent 架构；探索多模态" |

#### History 部分（历史上下文）

| 字段 | 时间范围 | 示例 |
|------|----------|------|
| `recentMonths` | 1-3个月 | "实现了记忆系统、优化了token计算" |
| `earlierContext` | 3-12个月 | "建立了项目架构、开发了技能系统" |
| `longTermBackground` | 长期背景 | "5年 Python 经验，专注于 AI 系统" |

#### Facts 部分（事实列表）

每条 fact 包含：

- **id**: 唯一标识符
- **content**: 具体内容
- **category**: 类别
  - `preference` - 偏好
  - `knowledge` - 知识/专长
  - `context` - 背景信息
  - `behavior` - 行为模式
  - `goal` - 目标
- **confidence**: 置信度（0.0-1.0）
- **createdAt**: 创建时间
- **source**: 来源线程

## 🔄 记忆更新流程

### 自动更新（推荐）

每次 Agent 执行完成后，系统会自动：

```
1. 对话完成
   ↓
2. 过滤消息（只保留用户输入和最终回复）
   ↓
3. 添加到更新队列
   ↓
4. 等待30秒（防抖，避免频繁调用）
   ↓
5. 调用 LLM 分析对话
   ↓
6. 提取新的 facts，更新 user/history 部分
   ↓
7. 保存到 memory.json 文件
```

**防抖机制**：
- 默认等待 30 秒后批量处理
- 在等待期间新增的更新会重置定时器
- 减少 LLM 调用次数，节省成本

### 手动更新

```python
from src.agents.memory import update_memory_from_conversation
from langchain_core.messages import HumanMessage, AIMessage

messages = [
    HumanMessage(content="我的项目使用 FastAPI"),
    AIMessage(content="FastAPI 是很好的选择...")
]

# 同步更新（立即调用 LLM）
success = update_memory_from_conversation(
    messages=messages,
    thread_id="thread_123",
    agent_name=None  # None=全局记忆，可指定agent名称
)
```

### 立即处理队列

```python
from src.agents.memory import get_memory_queue

queue = get_memory_queue()

# 查看队列状态
print(f"待处理: {queue.pending_count}")
print(f"处理中: {queue.is_processing}")

# 立即处理（跳过30秒等待）
queue.flush()
```

## 💡 记忆注入机制

### 如何注入？

在每次 LLM 调用前，系统会：

1. 提取最近 3 轮对话作为上下文
2. 基于上下文，使用 TF-IDF 计算相似度
3. 选择最相关的记忆内容（综合相似度和置信度）
4. 格式化为文本，注入到系统提示词的 `<memory>` 标签中

### 评分公式

```
最终得分 = 相似度 × 60% + 置信度 × 40%
```

### 手动格式化

```python
from src.agents.memory import format_memory_for_injection, get_memory_data

memory = get_memory_data()

# 格式化（最多2000 tokens）
memory_text = format_memory_for_injection(memory, max_tokens=2000)

# 输出示例：
# User Context:
# - Work: 后端工程师，主要项目 DeerFlow...
# - Personal: 双语能力（中文/英文）...
# - Current Focus: 正在学习 LangGraph...
# 
# History:
# - Recent: 过去3个月：实现了记忆系统...
```

### 在对话中使用

```python
from langchain_openai import ChatOpenAI
from langchain_core.prompts import ChatPromptTemplate

# 1. 获取并格式化记忆
memory = get_memory_data()
memory_text = format_memory_for_injection(memory, max_tokens=1500)

# 2. 构建系统提示词
system_prompt = f"""你是一个智能助手，了解用户的背景和偏好。

<memory>
{memory_text}
</memory>

请根据用户的记忆提供个性化的回复。"""

# 3. 调用 LLM
llm = ChatOpenAI(model="gpt-4")
prompt = ChatPromptTemplate.from_messages([
    ("system", system_prompt),
    ("human", "{input}")
])

chain = prompt | llm
response = chain.invoke({"input": "我该如何写测试？"})
# LLM 会基于记忆中 "偏好使用 pytest" 来回答
```

## ⚙️ 配置选项

### 完整配置

```yaml
memory:
  # 是否启用记忆功能
  enabled: true
  
  # 存储路径（空字符串=使用默认路径）
  # 支持绝对路径和相对路径（相对于 base_dir）
  storage_path: ""
  
  # 防抖延迟（秒）
  # 建议：频繁对话时增加，需要快速更新时减少
  debounce_seconds: 30
  
  # 用于记忆更新的模型名称
  # null = 使用默认模型
  model_name: null
  
  # 最大 facts 数量
  # 超出后保留置信度最高的前 N 条
  max_facts: 100
  
  # Facts 置信度阈值
  # 只保存置信度 >= 此值的 facts
  fact_confidence_threshold: 0.7
  
  # 是否启用记忆注入
  injection_enabled: true
  
  # 记忆注入的最大 token 数
  # 超出会自动截断
  max_injection_tokens: 2000
```

### Python 代码配置

```python
from src.config.memory_config import get_memory_config, set_memory_config, MemoryConfig

# 获取当前配置
config = get_memory_config()

new_config = MemoryConfig(
    enabled=True,
    debounce_seconds=60,  # 改为60秒
    max_facts=200,        # 增加到200条
    fact_confidence_threshold=0.8  # 提高阈值

set_memory_config(new_config)
```

### 场景3：记忆质量分析

```python
from src.agents.memory import get_memory_data

def analyze_memory():
    memory = get_memory_data()
    facts = memory["facts"]
    
    # 统计
    total = len(facts)
    by_category = {}
    total_confidence = 0
    
    for fact in facts:
        cat = fact["category"]
        by_category[cat] = by_category.get(cat, 0) + 1
        total_confidence += fact["confidence"]
    
    avg_confidence = total_confidence / total if total > 0 else 0
    
    print(f"📊 记忆质量分析")
    print(f"总事实数: {total}")
    print(f"平均置信度: {avg_confidence:.2%}")
    print(f"分类分布: {by_category}")

analyze_memory()
```

## ✅ 最佳实践

### 推荐做法

1. **记录具体信息**
   - ✅ "16k+ GitHub stars"
   - ❌ "很多star"

2. **保留专有名词**
   - ✅ "主要项目 DeerFlow，使用 LangGraph"
   - ❌ "主要项目某框架"

3. **设置合理置信度**
   - 明确陈述：0.9+
   - 强烈暗示：0.7-0.8
   - 推测：谨慎使用

4. **使用异步队列**
   - ✅ 让系统自动更新（默认行为）
   - ❌ 频繁同步调用 LLM

5. **定期审查**
   - 检查 memory.json 内容
   - 清理低质量 facts
   - 确保没有敏感信息

### 避免事项

1. **不要记录临时信息**
   - ❌ 文件上传路径（会话级）
   - ❌ 一次性的会话 ID

2. **不要使用过低置信度**
   - ❌ confidence < 0.7 的推测

3. **不要在多进程环境并发写入**
   - 当前使用文件存储，不支持并发
   - 建议单实例或迁移到数据库

4. **不要在对话中透露敏感信息**
   - 记忆系统会记录对话内容
   - 避免讨论密码、API Key 等

## ❓ 常见问题

### Q1: 记忆什么时候更新？

每次 Agent 执行完成后，自动添加到队列。默认等待 30 秒后批量处理。

### Q2: 如何立即更新？

```python
from src.agents.memory import get_memory_queue
get_memory_queue().flush()
```

### Q3: 如何清空记忆？

```bash
# 方法1：删除文件
rm /path/to/memory.json

# 方法2：Python 代码
from src.agents.memory.updater import _create_empty_memory, _save_memory_to_file
_save_memory_to_file(_create_empty_memory())
```

### Q4: 记忆会消耗多少 tokens？

默认最多 2000 tokens（可配置）。使用 tiktoken 精确计算。

### Q5: 支持多进程吗？

当前使用文件存储，**不支持**多进程并发写入。建议：
- 单实例部署
- 或迁移到数据库（PostgreSQL、Redis）

### Q6: 如何避免记录敏感信息？

- 定期审查 memory.json
- 在提示词中指示 LLM 不记录敏感内容
- 生产环境加密存储

### Q7: Per-agent 记忆是什么？

不同 agent 可以有独立的记忆空间：

```python
# 代码助手的记忆
code_memory = get_memory_data(agent_name="code_assistant")

# 聊天机器人的记忆
chat_memory = get_memory_data(agent_name="chat_bot")
```
